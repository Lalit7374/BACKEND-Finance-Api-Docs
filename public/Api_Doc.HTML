Quick Start
# 1. Make sure MongoDB is running
brew services start mongodb-community   # macOS
sudo systemctl start mongod             # Linux

# 2. Install & seed
npm install
npm run seed

# 3. Start server
npm start
# → http://localhost:3000
# → http://localhost:3000/docs  (this page)
Auth Flow

All endpoints except /api/auth/login require a Bearer token in the Authorization header.

Authorization: Bearer <your_jwt_token>
ℹTokens expire after 8 hours. Just log in again to get a new one.

Postman Testing Guide

Step-by-step setup

Follow these steps in order — the token from Step 1 is used in everything else.

1

Login to get your token

Send POST http://localhost:3000/api/auth/login with this body:

{
  "email":    "alice@demo.com",
  "password": "admin123"
}

Copy the token value from the response.

2

Create a Postman Environment

Top-right → Environments → Add
Add two variables:
  base_url = http://localhost:3000
  token = (paste the JWT here)

3

Set Authorization on every request

In each request → Authorization tab
Type: Bearer Token
Token: {{token}}

Or add a header manually:
Key: Authorization  ·  Value: Bearer {{token}}

4

Set Content-Type for POST / PATCH

Headers tab → add:
Key: Content-Type  ·  Value: application/json
Body tab → select raw → JSON

Standard Error Format

All errors follow this shape:

{
  "error":   "Short description",
  "details": ["field-level messages"]  // only on validation errors
}
StatusMeaning
200OK
201Resource created
400Bad request / validation failed
401Not authenticated / bad token
403Authenticated but not authorized (wrong role)
404Resource not found
409Conflict (e.g. email already in use)
500Server error
